Per-project PHP version routing.

Pin a PHP version in your composer.json. composer and php automatically run on that version inside the project — and fall back to your system default outside.

Like Volta, but for PHP. Works on Linux, macOS, WSL, and Windows.

Install View on GitHub

Install

curl -fsSL https://raw.githubusercontent.com/phpick/phpick/main/install.sh | bash

Drops a single shim at ~/.phpick/bin/phpick, symlinks it as composer and php, and puts ~/.phpick/bin first on your PATH. Open a new shell and you're done. 

irm https://raw.githubusercontent.com/phpick/phpick/main/install.ps1 | iex

Downloads phpick.exe (Rust native, ~300 KB) to %LOCALAPPDATA%\phpick\bin, copies it as php.exe and composer.exe, and prepends that directory to your user PATH. Auto-detects PHP from Laragon, Scoop, Chocolatey, and Herd.

Pin a project

Use composer's built-in config:

composer config platform.php 8.4.15

That writes to your composer.json:

{
  "config": {
    "platform": {
      "php": "8.4.15"
    }
  }
}

phpick reads the major.minor (8.4) and routes to a matching installed PHP binary. Bonus: composer already uses config.platform.php for dependency resolution, so pinning here keeps both runtime and lockfile in sync.

How it works

When you run composer or php, the shim:

  1. Walks up from $PWD to the nearest composer.json.
  2. Reads config.platform.php.
  3. Resolves the matching PHP binary in this order:
    • phpX.Y on PATH — Debian / Ubuntu
    • phpXY on PATH — some RHEL-family distros
    • Homebrew — brew --prefix php@X.Y
    • asdf — ~/.asdf/installs/php/X.Y.*/bin/php
    • phpenv — ~/.phpenv/versions/X.Y.*/bin/php
    • Laravel Herd — ~/Library/Application Support/Herd/bin/phpX.Y
    • Laragon (Windows) — C:\laragon\bin\php\php-X.Y.*\php.exe
    • Scoop / Chocolatey / generic C:\Program Files\PHP\X.Y
  4. Execs the real composer or php under that PHP.

No pin, no composer.json, or the pinned version isn't installed → falls back to your system default with a warning in the last case.

Updating

phpick check-update    # is a newer version published?
phpick update          # reinstall in place

The shim also runs a quiet background check once per UTC day when you invoke php or composer interactively, and prints a single-line hint to stderr if a newer version is published. Disable with PHPICK_NO_UPDATE_CHECK=1.
